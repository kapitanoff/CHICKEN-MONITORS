# Chicken Monitor

Chicken Monitor - это веб-панель для контроля температуры куриц по беспроводным датчикам.

Приложение получает данные от датчиков через MQTT, сохраняет историю в базу данных и показывает состояние каждой курицы в браузере: температуру, статус, заряд батареи, загон и график изменения температуры.

Репозиторий проекта:

```text
https://github.com/kapitanoff/CHICKEN-MONITORS.git
```

Короткая памятка для переноса на ПК фермы, обновления и тестовых данных лежит в файле:

```text
FARM_TRANSFER_INSTRUCTIONS.txt
```

## Самое важное

После установки панель открывается в браузере:

```text
http://localhost:8000
```

`localhost` означает "этот компьютер". Если открыть этот адрес на другом компьютере или телефоне, он не сработает. Для доступа с другого устройства в сети нужна отдельная настройка сети и Docker.

Цвет и подпись на карточке показывают состояние:

| Статус | Что значит | Что делать |
|---|---|---|
| Норма | Температура в нормальном диапазоне | Ничего срочного. |
| Предупреждение | Температура выше нормы, но еще не критическая | Проверить при ближайшем обходе. |
| Опасно | Температура слишком низкая или слишком высокая | Проверить курицу сразу. |
| Нет данных | От датчика нет свежих данных | Проверить датчик, батарейку и связь. |

Стандартные пороги:

- `40.0-42.0 °C` - норма;
- `42.0-43.0 °C` - предупреждение;
- ниже `40.0 °C` или выше `43.0 °C` - опасно.

## Ежедневное использование

Обычный порядок такой:

1. Включите компьютер.
2. Запустите Docker Desktop, если он не запустился сам.
3. Подождите, пока Docker Desktop полностью откроется.
4. Откройте браузер.
5. Перейдите по адресу:

```text
http://localhost:8000
```

Если страница открылась и карточки обновляются, больше ничего запускать не нужно.

Если после перезагрузки компьютера страница не открывается, откройте PowerShell в папке проекта и выполните:

```powershell
docker compose up -d
```

`setup.ps1` не нужно запускать каждый день. Он нужен только для первой настройки или если нужно заново поменять основные настройки подключения.

## Что нужно подготовить перед установкой

Перед установкой нужен человек, который знает данные датчиков и MQTT-брокера.

Подготовьте:

1. Компьютер с Windows 10/11, macOS или Linux.
2. Доступ в интернет для первой установки.
3. Docker Desktop для Windows/macOS или Docker Engine для Linux.
4. Папку с проектом Chicken Monitor.
5. Адрес MQTT-брокера.
6. Порт MQTT-брокера. Обычно это `1883`.
7. Логин и пароль MQTT, если они используются.
8. Понимание, какие пороги температуры считать нормой.

Git нужен только в двух случаях: если проект скачивают командой `git clone` или если обновляют его командой `git pull`. Если проект передали ZIP-архивом, Git для обычного запуска не нужен.

MQTT-брокер - это программа или устройство, куда датчики отправляют данные. Chicken Monitor сам не опрашивает датчики напрямую. Он читает данные из MQTT.

Если вы не знаете адрес, логин или пароль MQTT, не угадывайте. Их должен дать человек, который настраивал датчики.

## Установка на Windows 10/11

Это основной способ установки.

### 1. Установите Docker Desktop

1. Откройте официальную страницу Docker Desktop:
   https://www.docker.com/products/docker-desktop/
2. Скачайте Docker Desktop для Windows.
3. Установите его как обычную программу.
4. Если установщик попросит перезагрузить компьютер, перезагрузите.
5. Запустите Docker Desktop.
6. Дождитесь, пока Docker полностью запустится.

Обычно значок Docker появляется рядом с часами в правом нижнем углу экрана.

Если Docker пишет про WSL 2 или виртуализацию, следуйте подсказкам установщика. Без этого Docker Desktop на Windows обычно не работает.

### 2. Скачайте проект

Если проект скачивают через Git:

```powershell
git clone https://github.com/kapitanoff/CHICKEN-MONITORS.git
cd CHICKEN-MONITORS
```

Если проект передали ZIP-архивом:

1. Распакуйте архив.
2. Откройте распакованную папку.
3. Не удаляйте файл `.env`, если он уже есть. В нем лежат настройки подключения.

### 3. Откройте PowerShell в папке проекта

Самый простой способ:

1. Откройте папку проекта в Проводнике.
2. Нажмите правой кнопкой мыши по пустому месту внутри папки.
3. Выберите `Открыть в терминале` или `Открыть окно PowerShell здесь`.

Если такого пункта нет:

1. Нажмите на адресную строку Проводника.
2. Скопируйте путь к папке.
3. Откройте PowerShell через меню `Пуск`.
4. Введите `cd `, вставьте путь и нажмите Enter.

Пример:

```powershell
cd C:\Users\Ivan\Desktop\CHICKEN-MONITORS
```

### 4. Запустите настройку

В PowerShell выполните:

```powershell
.\setup.ps1
```

Скрипт задаст вопросы и создаст файл `.env`.

| Вопрос | Что вводить |
|---|---|
| `MQTT broker IP address` | Адрес MQTT-брокера. Например `192.168.1.50`. |
| `MQTT port [1883]` | Обычно просто нажмите Enter. |
| `MQTT username` | Логин MQTT. Если логина нет, нажмите Enter. |
| `MQTT password` | Пароль MQTT. Если пароля нет, нажмите Enter. |
| `PostgreSQL password [auto-generate]` | Обычно просто нажмите Enter. |
| `Normal min [40.0]` | Нижняя граница нормы. Обычно Enter. |
| `Normal max [42.0]` | Верхняя граница нормы. Обычно Enter. |
| `Warning max [43.0]` | Верхняя граница предупреждения. Обычно Enter. |
| `Start Chicken Monitor now? (Y/n)` | Нажмите Enter. |

Первый запуск может занять несколько минут. Docker скачивает и собирает нужные части приложения.

Если вы случайно запустили `setup.ps1` повторно и он спрашивает, перезаписать ли `.env`, обычно отвечайте `N`. Так старые настройки MQTT и базы данных останутся на месте.

### 5. Откройте панель

Откройте в браузере:

```text
http://localhost:8000
```

Если датчики уже отправляют данные, появятся карточки.

## Если PowerShell запрещает запуск скрипта

На Windows может появиться ошибка:

```text
running scripts is disabled on this system
```

или похожий текст на русском.

Запустите настройку с временным разрешением:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

Если Windows пишет, что файл заблокирован после скачивания из интернета:

```powershell
Unblock-File .\setup.ps1
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

## Как пользоваться экраном

### Карточки

Одна карточка - это одна курица или один датчик.

На карточке видно:

- номер курицы или датчика;
- текущую температуру;
- значок батареи;
- статус: `Норма`, `Предупреждение`, `Опасно` или `Нет данных`;
- загон, если курица назначена в загон.

Цветная полоска сверху карточки помогает быстро увидеть проблему. Красный статус важнее желтого, желтый важнее зеленого.

### Счетчики сверху

Сверху есть счетчики:

- `Опасно`;
- `Предупреждение`;
- `Норма`.

Нажмите на счетчик, чтобы показать только такие карточки. Например, если нажать `Опасно`, на экране останутся только опасные датчики.

Чтобы снова увидеть всех, нажмите `Показать все`.

### Режимы просмотра

Вверху есть два режима:

| Кнопка | Что показывает |
|---|---|
| `Все` | Все курицы общим списком. |
| `По загонам` | Куриц, разделенных по загонам. |

В режиме `Все` можно выбрать, сколько ячеек показывать на странице: `40` или `60`.

Обычно удобно:

- `40` - если экран небольшой или нужно крупнее;
- `60` - если экран широкий и нужно видеть больше куриц сразу.

В режиме `По загонам` карточки сгруппированы по загонам. Загон можно свернуть или раскрыть нажатием на его заголовок.

### График температуры

Чтобы открыть историю температуры:

1. Нажмите на карточку курицы.
2. Откроется окно с графиком.
3. Выберите период: `1ч`, `6ч`, `24ч`, `7д`, `30д`, `6мес`, `1год`.

В этом же окне можно изменить загон курицы.

### Удаление курицы

В окне карточки есть кнопка удаления.

Удаление убирает курицу и ее историю из системы. Не нажимайте `Удалить`, если не уверены.

Если датчик продолжит отправлять данные после удаления, курица может появиться снова как новая.

## Как работать с загонами

Загоны нужны, чтобы разделить куриц по корпусам, клеткам, помещениям или другим группам.

### Создать загон

1. Нажмите `Управление загонами`.
2. Введите название. Например `Корпус 1` или `Загон 3`.
3. Нажмите `Создать`.

### Посмотреть куриц по загонам

1. Нажмите `По загонам`.

### Назначить одну курицу в загон

1. Нажмите на карточку курицы.
2. В поле `Загон` выберите нужный загон.
3. Закройте окно.

### Назначить несколько куриц сразу

Есть два способа включить выбор нескольких куриц.

Способ 1 - через кнопку:

1. Нажмите `Выбрать`.
2. Отметьте нужные карточки.
3. В панели выбора выберите загон.
4. Нажмите `Назначить`.
5. Нажмите `Отмена`, чтобы выйти из режима выбора.

Способ 2 - через карточку:

1. В режиме `Все` зажмите карточку курицы примерно на полсекунды.
2. Система включит режим выбора и отметит эту курицу.
3. Нажмите на остальные карточки, которые нужно добавить в выбор.
4. В панели выбора выберите загон.
5. Нажмите `Назначить`.
6. Нажмите `Отмена`, чтобы выйти из режима выбора.

После нажатия `Выбрать` в режиме `По загонам` можно выбрать весь загон через чекбокс в заголовке загона.

## Пороги температуры

Пороги определяют, когда карточка будет нормальной, предупреждающей или опасной.

Открыть настройки порогов:

1. Нажмите значок шестеренки рядом с легендой цветов.
2. Измените значения.
3. Нажмите `Сохранить`.

Поля означают:

| Поле | Что значит |
|---|---|
| `Норма от` | Если температура ниже, статус станет `Опасно`. |
| `Норма до` | Если температура выше, статус станет `Предупреждение`. |
| `Предупреждение до` | Если температура выше, статус станет `Опасно`. |

Важно: изменение порогов через экран работает до перезапуска контейнера. Чтобы сохранить пороги постоянно, измените эти же значения в файле `.env` и перезапустите систему.

## Запуск, остановка, перезапуск и обновление

Все команды ниже выполняются в PowerShell или терминале в папке проекта.

### Запустить

```powershell
docker compose up -d
```

### Остановить без удаления данных

```powershell
docker compose down
```

История температур при этом сохраняется.

### Перезапустить

```powershell
docker compose restart
```

Если после изменения `.env` нужно перечитать настройки, лучше выполнить:

```powershell
docker compose down
docker compose up -d
```

### Пересобрать после изменения файлов

```powershell
docker compose up -d --build
```

### Проверить, работает ли система

```powershell
docker compose ps
```

Нормально, если контейнеры находятся в состоянии `Up` или `healthy`.

### Посмотреть логи

```powershell
docker compose logs backend
```

Если логов слишком много:

```powershell
docker compose logs --tail=100 backend
```

### Обновить проект через Git

Если проект был скачан через `git clone`:

```powershell
git pull
docker compose up -d --build
```

### Обновить проект из ZIP-архива

1. Остановите систему:

```powershell
docker compose down
```

2. Замените файлы проекта новыми.
3. Не удаляйте `.env`.
4. Запустите:

```powershell

```

### Полностью удалить данные и начать заново

```powershell
docker compose down -v
```

Осторожно: эта команда удаляет базу данных и всю историю температур.

## Тестовые данные для показа интерфейса

Если нужно показать экран с квадратными ячейками `40 / 60`, можно добавить демо-датчики.

Команда добавляет 100 тестовых куриц `TEST001`-`TEST100` и два демо-загона. Реальные курицы не удаляются.

```powershell
.\test-data.ps1
```

После этого откройте:

```text
http://localhost:8000
```

И нажмите `Ctrl + F5`.

Удалить только тестовые данные:

```powershell
.\test-data.ps1 -Action clear
```

Добавить другое количество тестовых куриц:

```powershell
.\test-data.ps1 -Count 60
```

Скрипт работает только с курицами, у которых ID начинается на `TEST`, и с группами `Демо загон 1`, `Демо загон 2`, `Test sensors`.

## Настройки `.env`

Файл `.env` находится в папке проекта. Его создает `setup.ps1` или `setup.sh`.

Не отправляйте `.env` посторонним людям: в нем могут быть пароли.

Основные настройки:

| Параметр | Что значит | Пример |
|---|---|---|
| `MQTT_HOST` | Адрес MQTT-брокера | `192.168.1.50` |
| `MQTT_PORT` | Порт MQTT-брокера | `1883` |
| `MQTT_USERNAME` | Логин MQTT | `chicken` |
| `MQTT_PASSWORD` | Пароль MQTT | `secret` |
| `MQTT_QOS` | Надежность доставки MQTT | `0`, `1` или `2` |
| `POSTGRES_PASSWORD` | Внутренний пароль базы данных | создается скриптом |
| `TEMP_GREEN_MIN` | Нижняя граница нормы | `40.0` |
| `TEMP_GREEN_MAX` | Верхняя граница нормы | `42.0` |
| `TEMP_YELLOW_MAX` | Верхняя граница предупреждения | `43.0` |

После изменения `.env` перезапустите систему:

```powershell
docker compose down
docker compose up -d
```

## Если что-то не работает

### Страница `http://localhost:8000` не открывается

Проверьте по порядку:

1. Docker Desktop запущен.
2. Вы открываете адрес на том же компьютере, где запущен проект.
3. PowerShell открыт в папке проекта.
4. Система запущена:

```powershell
docker compose ps
```

Если контейнеры не запущены:

```powershell
docker compose up -d
```

### PowerShell пишет, что `docker` не найден

Возможные причины:

1. Docker Desktop не установлен.
2. Docker Desktop установлен, но еще не запущен.
3. PowerShell был открыт до установки Docker.

Что сделать:

1. Запустите Docker Desktop.
2. Закройте PowerShell.
3. Откройте PowerShell заново в папке проекта.
4. Выполните:

```powershell
docker version
```

### На странице написано `Нет связи с сервером`

1. Подождите 10-20 секунд.
2. Проверьте Docker:

```powershell
docker compose ps
```

3. Если контейнеры остановлены, запустите:

```powershell
docker compose up -d
```

4. Если ошибка остается, посмотрите логи:

```powershell
docker compose logs --tail=100 backend
```

### Карточки не появляются

Проверьте:

1. Датчики включены.
2. Датчики отправляют данные в MQTT.
3. В `.env` указан правильный `MQTT_HOST`.
4. В `.env` указан правильный `MQTT_PORT`.
5. Логин и пароль MQTT указаны правильно, если они нужны.
6. Топики датчиков совпадают с форматом из раздела [Подключение датчиков](#подключение-датчиков).

Посмотреть ошибки MQTT:

```powershell
docker compose logs --tail=100 backend
```

Если в логах есть `MQTT connection failed` или `MQTT reconnect failed`, проблема обычно в адресе MQTT-брокера, порте, логине, пароле или доступности сети.

### Порт 8000 занят

Если Docker пишет, что порт `8000` занят, значит другая программа уже использует этот порт.

Попробуйте:

```powershell
docker compose down
docker compose up -d
```

Если ошибка остается, попросите технического специалиста проверить, какая программа занимает порт.

### После перезагрузки компьютера ничего не работает

1. Запустите Docker Desktop.
2. Подождите, пока он полностью загрузится.
3. Откройте PowerShell в папке проекта.
4. Выполните:

```powershell
docker compose up -d
```

## Подключение датчиков

Этот раздел нужен человеку, который настраивает датчики.

Датчики должны отправлять простое число в MQTT.

Топик температуры:

```text
ThermoChicken/<номер_датчика>/Temperature
```

Топик напряжения батареи:

```text
ThermoChicken/<номер_датчика>/voltage
```

Пример температуры:

```text
41.2
```

Пример напряжения батареи:

```text
2.95
```

Номер датчика может содержать латинские буквы, цифры, `_` и `-`. Максимальная длина - 32 символа.

## Windows 7

Docker Desktop не работает на Windows 7. Для Windows 7 в проекте есть вариант запуска через VirtualBox.

Это более сложный вариант. Его лучше выполнять человеку, который уверенно работает с компьютером.

Понадобится:

- VirtualBox 6.1;
- Git for Windows;
- доступ в интернет;
- права администратора.

Короткий порядок:

1. В корне проекта выполните:

```powershell
.\setup-win7.ps1
```

2. Перейдите в папку `vm`:

```powershell
cd vm
```

3. Запустите установку виртуальной машины от имени администратора:

```powershell
.\deploy-vbox.ps1
```

После завершения откройте:

```text
http://localhost:8080
```

Обновление для VirtualBox-варианта:

```powershell
cd vm
.\update-vbox.ps1
```

Удаление базы в VirtualBox-варианте:

```powershell
cd vm
.\update-vbox.ps1 -ResetDB
```

Осторожно: `-ResetDB` удаляет историю температур.

## Что не стоит делать без понимания

Не делайте это без необходимости:

- не удаляйте файл `.env`;
- не удаляйте папку проекта, если не сделали копию `.env`;
- не выполняйте `docker compose down -v`, если нужно сохранить историю;
- не меняйте `docker-compose.yml`, если не знаете, зачем;
- не открывайте доступ к панели с другого компьютера без настройки безопасности сети.

## Короткая памятка команд

| Действие | Команда |
|---|---|
| Запустить | `docker compose up -d` |
| Остановить | `docker compose down` |
| Перезапустить | `docker compose restart` |
| Пересобрать | `docker compose up -d --build` |
| Проверить состояние | `docker compose ps` |
| Посмотреть логи | `docker compose logs --tail=100 backend` |
| Обновить через Git | `git pull`, затем `docker compose up -d --build` |
| Удалить все данные | `docker compose down -v` |
