# Chicken Monitor

Chicken Monitor - это веб-панель для контроля температуры куриц по беспроводным датчикам.

Приложение получает данные из MQTT, сохраняет историю и показывает состояние каждой курицы в браузере: карточки, цветовые статусы, заряд батареи, загоны и графики.

Репозиторий проекта:

```text
https://github.com/kapitanoff/CHICKEN-MONITORS.git
```

## Самое главное

После установки приложение открывается в браузере:

```text
http://localhost:8000
```

Если все работает, на экране появятся карточки куриц.

Цвет карточки показывает состояние:

| Цвет | Значение | Что делать |
|---|---|---|
| Зеленый | Температура в норме | Ничего срочного. |
| Желтый | Температура выше нормы | Проверить курицу при ближайшем обходе. |
| Красный | Температура слишком низкая или слишком высокая | Проверить курицу сразу. |
| Серый или прочерк | Нет свежих данных от датчика | Проверить датчик, батарейку и связь. |

Стандартные пороги:

- норма: от `40.0` до `42.0 °C`;
- предупреждение: выше `42.0` до `43.0 °C`;
- опасность: ниже `40.0` или выше `43.0 °C`.

## Быстрый запуск

Коротко для тех, кто уже понимает, что делает:

```powershell
git clone https://github.com/kapitanoff/CHICKEN-MONITORS.git
cd CHICKEN-MONITORS
.\setup.ps1
```

Потом откройте:

```text
http://localhost:8000
```

Если PowerShell не разрешает запустить `setup.ps1`, смотрите раздел [Если PowerShell запрещает запуск скрипта](#если-powershell-запрещает-запуск-скрипта).

## Что понадобится

Для обычного просмотра нужен только браузер и адрес страницы.

Для установки нужны:

1. Компьютер с Windows 10/11, macOS или Linux.
2. Docker Desktop для Windows/macOS или Docker Engine для Linux.
3. MQTT-брокер, к которому подключены датчики.
4. Адрес, порт, логин и пароль MQTT-брокера, если они используются.

MQTT-брокер - это место, куда датчики отправляют данные. Chicken Monitor не заменяет датчики и не заменяет MQTT-брокер. Он только читает готовые данные и показывает их на экране.

Если вы не знаете данные MQTT-брокера, не угадывайте. Их должен дать человек, который настраивал датчики.

## Установка на Windows 10/11

### Шаг 1. Установите Docker Desktop

1. Откройте страницу Docker Desktop для Windows:
   https://docs.docker.com/desktop/setup/install/windows-install/
2. Скачайте установщик.
3. Установите Docker как обычную программу.
4. Перезагрузите компьютер.
5. Запустите Docker Desktop.
6. Дождитесь, пока Docker полностью откроется.

Обычно значок Docker появляется рядом с часами в правом нижнем углу экрана.

### Шаг 2. Скачайте проект

Если скачиваете через Git:

```powershell
git clone https://github.com/kapitanoff/CHICKEN-MONITORS.git
cd CHICKEN-MONITORS
```

Если проект передали ZIP-архивом:

1. Распакуйте архив.
2. Откройте распакованную папку проекта.

### Шаг 3. Откройте PowerShell в папке проекта

Самый простой способ:

1. Откройте папку проекта в Проводнике.
2. Нажмите правой кнопкой мыши по пустому месту внутри папки.
3. Выберите `Открыть в терминале` или `Открыть окно PowerShell здесь`.

Если такого пункта нет:

1. Нажмите на адресную строку Проводника.
2. Скопируйте путь к папке.
3. Откройте PowerShell через меню `Пуск`.
4. Введите `cd `, вставьте путь и нажмите Enter.

Пример:

```powershell
cd C:\Users\Иван\Desktop\CHICKEN-MONITORS
```

### Шаг 4. Запустите настройку

В PowerShell выполните:

```powershell
.\setup.ps1
```

Скрипт задаст вопросы и создаст файл `.env` с настройками.

| Вопрос | Что вводить |
|---|---|
| `MQTT broker IP address` | Адрес MQTT-брокера. Например `localhost` или `192.168.1.50`. |
| `MQTT port [1883]` | Обычно просто нажмите Enter. |
| `MQTT username` | Логин MQTT. Если логина нет, нажмите Enter. |
| `MQTT password` | Пароль MQTT. Если пароля нет, нажмите Enter. |
| `PostgreSQL password [auto-generate]` | Просто нажмите Enter. |
| `Normal min [40.0]` | Просто нажмите Enter, если подходит стандартная норма. |
| `Normal max [42.0]` | Просто нажмите Enter. |
| `Warning max [43.0]` | Просто нажмите Enter. |
| `Start Chicken Monitor now? (Y/n)` | Нажмите Enter. |

Первый запуск может занять несколько минут: Docker скачивает и собирает нужные части приложения.

### Шаг 5. Откройте приложение

Откройте в браузере:

```text
http://localhost:8000
```

Если датчики уже отправляют данные, появятся карточки куриц.

## Если PowerShell запрещает запуск скрипта

На Windows может появиться ошибка:

```text
Невозможно загрузить файл ... setup.ps1, так как выполнение сценариев отключено в этой системе.
```

или:

```text
running scripts is disabled on this system
```

Это настройка безопасности Windows. Проект не сломан.

Самый простой вариант - запустить настройку один раз с временным разрешением:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

Если Windows пишет, что файл заблокирован после скачивания из интернета:

```powershell
Unblock-File .\setup.ps1
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

Если хотите разрешить запуск локальных PowerShell-скриптов для своего пользователя:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
.\setup.ps1
```

Когда PowerShell спросит подтверждение, нажмите `Y`, затем Enter.

## Как пользоваться

На главном экране каждая карточка - это одна курица или один датчик.

Карточка показывает:

- номер курицы;
- текущую температуру;
- заряд батареи датчика;
- цветовой статус;
- загон, если курица назначена в группу.

Сверху есть счетчики `Опасно`, `Предупреждение`, `Норма`. Нажмите на счетчик, чтобы показать только таких куриц. Нажмите `Показать все`, чтобы убрать фильтр.

Чтобы открыть график, нажмите на карточку курицы.

В окне графика можно выбрать период:

- `1ч`;
- `6ч`;
- `24ч`;
- `7д`;
- `30д`;
- `6мес`;
- `1год`.

В этом же окне можно назначить курицу в загон или удалить ее из системы.

Не нажимайте `Удалить`, если не уверены. Удаление убирает курицу и ее историю из системы.

## Загоны

Загоны нужны, чтобы разделить куриц по корпусам, клеткам или другим группам.

Создать загон:

1. Нажмите `Управление загонами`.
2. Введите название, например `Корпус 1`.
3. Нажмите `Создать`.

Показать куриц по загонам:

1. Нажмите `По загонам` вверху экрана.

Назначить одну курицу в загон:

1. Нажмите на карточку курицы.
2. В поле `Загон` выберите нужный загон.

Назначить несколько куриц сразу:

1. Нажмите `Выбрать`.
2. Отметьте нужные карточки.
3. Выберите загон в панели снизу.
4. Нажмите `Назначить`.

## Пороги температуры

Пороги температуры определяют, когда карточка зеленая, желтая или красная.

Изменить пороги можно через значок шестеренки рядом с цветовой легендой.

Поля означают:

- `Норма от` - если температура ниже, карточка станет красной;
- `Норма до` - если температура выше, карточка станет желтой;
- `Предупреждение до` - если температура выше, карточка станет красной.

После изменения нажмите `Сохранить`.

Важно: изменение через интерфейс действует до перезапуска контейнера. Чтобы сохранить значения постоянно, измените эти же параметры в файле `.env` и перезапустите систему.

## Если что-то не работает

### Страница не открывается

1. Проверьте, что Docker Desktop запущен.
2. Откройте PowerShell в папке проекта.
3. Выполните:

```powershell
docker compose ps
```

Контейнеры должны быть в состоянии `running` или `healthy`.

Если видите ошибку, посмотрите логи:

```powershell
docker compose logs backend
```

### На странице написано "Нет связи с сервером"

1. Подождите 10-20 секунд.
2. Проверьте, что Docker Desktop запущен.
3. Проверьте, что приложение запущено:

```powershell
docker compose ps
```

Если контейнеры не запущены:

```powershell
docker compose up -d
```

### Карточки куриц не появились

Проверьте по порядку:

1. Датчики включены.
2. Датчики подключены к правильному MQTT-брокеру.
3. В `.env` указан правильный `MQTT_HOST`.
4. В `.env` указаны правильные `MQTT_USERNAME` и `MQTT_PASSWORD`, если они нужны.
5. Датчики отправляют данные именно в топики `ThermoChicken/<номер>/Temperature` и `ThermoChicken/<номер>/voltage`.

Посмотреть ошибки подключения к MQTT:

```powershell
docker compose logs backend
```

Если в логах есть `MQTT connection failed` или `MQTT reconnect failed`, проблема обычно в адресе MQTT-брокера, порте, логине или пароле.

### Docker пишет, что порт 8000 занят

Значит на компьютере уже работает другая программа на этом порту.

Самый простой вариант:

1. Закройте другую программу, если знаете что это.
2. Перезапустите Chicken Monitor:

```powershell
docker compose down
docker compose up -d
```

Если не знаете, что занимает порт, попросите помощи у человека, который обслуживает компьютер.

## Подключение датчиков

Датчики должны отправлять простое число в MQTT.

Топик температуры:

```text
ThermoChicken/<номер_датчика>/Temperature
```

Топик напряжения батареи:

```text
ThermoChicken/<номер_датчика>/voltage
```

Пример температуры:

```text
41.2
```

Пример напряжения батареи:

```text
2.95
```

Номер датчика может содержать латинские буквы, цифры, `_` и `-`. Максимальная длина - 32 символа.

## Запуск, остановка и обновление

Остановить систему без удаления данных:

```powershell
docker compose down
```

Запустить снова:

```powershell
docker compose up -d
```

Пересобрать и запустить после обновления файлов:

```powershell
docker compose up -d --build
```

Обновить проект, если он скачан через Git:

```powershell
git pull
docker compose up -d --build
```

Полностью удалить базу данных и начать заново:

```powershell
docker compose down -v
```

Осторожно: `docker compose down -v` удаляет всю историю температур.

## Настройки .env

Файл `.env` находится в папке проекта. Его создает `setup.ps1` или `setup.sh`.

Главные настройки:

| Параметр | Что значит | Пример |
|---|---|---|
| `MQTT_HOST` | Адрес MQTT-брокера | `localhost` или `192.168.1.50` |
| `MQTT_PORT` | Порт MQTT-брокера | `1883` |
| `MQTT_USERNAME` | Логин MQTT | `chicken` |
| `MQTT_PASSWORD` | Пароль MQTT | `secret` |
| `MQTT_QOS` | Надежность доставки MQTT | `0`, `1` или `2` |
| `POSTGRES_PASSWORD` | Внутренний пароль базы данных | создается скриптом |
| `TEMP_GREEN_MIN` | Нижняя граница нормы | `40.0` |
| `TEMP_GREEN_MAX` | Верхняя граница нормы | `42.0` |
| `TEMP_YELLOW_MAX` | Верхняя граница предупреждения | `43.0` |

После ручного изменения `.env` перезапустите систему:

```powershell
docker compose down
docker compose up -d
```

## Windows 7

Docker Desktop не работает на Windows 7. Для Windows 7 в проекте есть вариант запуска через VirtualBox.

Короткий порядок:

1. Установите VirtualBox 6.1.
2. Установите Git for Windows.
3. В папке проекта выполните:

```powershell
.\setup-win7.ps1
```

4. Перейдите в папку `vm`:

```powershell
cd vm
```

5. Запустите установку виртуальной машины от имени администратора:

```powershell
.\deploy-vbox.ps1
```

После завершения откройте:

```text
http://localhost:8080
```

Это более сложный вариант. Если есть возможность использовать Windows 10/11 с Docker Desktop, лучше использовать обычную установку выше.
